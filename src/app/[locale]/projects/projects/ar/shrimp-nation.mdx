## نظرة عامة

شريمب نيشن منصة توصيل طعام متكاملة مصممة للسوق الليبي. تربط المنصة العملاء بالمطاعم والسائقين وخدمات التوصيل من خلال واجهة Laravel API موحدة. يتكون النظام من 5 تطبيقات Flutter (العميل، السائق، البائع، نقطة البيع، النادل)، موقع Next.js للعملاء، لوحة إدارة React لإدارة الأعمال، وصفحة QR بـ Vite/React للطلب من الطاولة. تدعم جميع المكونات العربية (RTL)، الدفع بالدينار الليبي، وتكاملات محلية مثل معاملات باي و iSend SMS.

## المميزات الرئيسية

- تطبيق العميل (Flutter): تصفح المطاعم والقوائم، سلة التسوق، الدفع، تتبع الطلبات، خرائط جوجل، التقييمات، Firebase Auth، ودعم العربية والإنجليزية.
- تطبيق السائق (Flutter): قبول الطلبات، تدفق التوصيل، تتبع GPS، الأرباح، إشعارات الدفع، والعمل في الخلفية عبر WorkManager.
- تطبيق البائع (Flutter): إدارة المطاعم—القائمة، الطلبات، التحليلات، الخرائط، ومناطق التوصيل.
- تطبيق نقطة البيع (Flutter): نقطة بيع سطح المكتب لـ Windows مع طباعة الإيصالات (ESC/POS)، إدارة الطاولات، واكتشاف الطابعات الشبكية.
- تطبيق النادل (Flutter): أخذ طلبات الطاولات وإرسالها مع خرائط OSM.
- الموقع (Next.js): سوق إلكتروني للعملاء مع تصفح المطاعم، السلة، الدفع، i18n، و Firebase.
- لوحة الإدارة (React + Ant Design): إدارة المطاعم والطلبات والمستخدمين والتوصيل والمالية مع RTL وتكامل iSend SMS.
- صفحة QR (Vite + React): صفحات طلب خاصة بكل طاولة قابلة للنشر على Vercel.

## التقنيات المستخدمة

- Laravel 8: REST API، مصادقة Sanctum، صلاحيات Spatie، MySQL، Redis، S3، متعدد اللغات والعملات.
- Flutter 3.24+: Riverpod، Auto Route، Dio، Firebase (Auth، Messaging، Firestore)، خرائط جوجل، OSM.
- Next.js 13: موقع العملاء مع react-i18next، Redux، React Query.
- React 18 + Ant Design 5: لوحة الإدارة مع Redux Toolkit و SCSS RTL.
- Vite + React: صفحة QR.
- التكاملات: معاملات باي، iSend SMS، Firebase.

## التحديات والدروس المستفادة

بناء نظام متعدد التطبيقات مع 9 تطبيقات تطلّب تصميم API قوي وموثق جيداً بحيث تشارك تطبيقات Flutter والمواقع ولوحة الإدارة وصفحات QR نفس خلفية Laravel. تنفيذ دعم كامل للعربية و RTL عبر React و Flutter و Laravel—بما في ذلك خط Cairo والتبديل الديناميكي للغة—علّمني أهمية أنماط الترجمة المتسقة. تكامل الخدمات الليبية (معاملات باي، iSend SMS) تطلّب التكيف مع واجهات API المحلية وتدفقات الدفع. إدارة مزامنة الطلبات في الوقت الفعلي بين واجهات العميل والسائق والبائع والإدارة كانت تحدياً رئيسياً.

## النتيجة

شريمب نيشن تقدم منصة توصيل طعام متكاملة مصممة خصيصاً لإحتياجات المطعم: العملاء يمكنهم الطلب عبر الويب أو الموبايل، المطاعم تدير العمليات عبر تطبيق البائع ولوحة الإدارة، السائقون يتعاملون مع التوصيلات مع تتبع مخصص، والزبائن داخل المطعم يمكنهم الطلب عبر QR. يدعم النظام الدفع بالدينار الليبي والعربية و RTL والرسائل القصيرة المحلية—ما يجعله جاهزاً للإنتاج في السوق الليبي.
